# レジリエンス: 再接続とキープアライブ

クライアントは、一時的な切断の後に MPQUIC トンネルを自動的に再確立します。切断時には指数バックオフのリトライループ（`--reconnect-max-backoff` で上限、雪崩を避けるためジッタ付き）に入り、トンネルが復帰するまで無期限にリトライします — プロセスの再起動なしに、ローカルの SOCKS5・HTTP CONNECT・ゲートウェイの各リスナーに触れることもありません。アイドル状態のトンネルは定期的な QUIC PING（`--keepalive-idle`）で維持されます。再接続はデフォルト有効で、`--no-reconnect` で無効化できます。

::: warning 制限
完全な接続喪失の瞬間に進行中だったフローは失敗します — 復旧後に復活はしません。トンネル復帰後に開かれた新しいフローは自動的に機能します。
:::

クライアントフラグ `--reconnect` / `--no-reconnect`、`--reconnect-max-backoff`、`--keepalive-idle` については [オプションリファレンス](/ja/reference/options#共通フラグ-全モード) を参照してください。
